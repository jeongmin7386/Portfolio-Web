# Studiofflower.dev 사용법

이 문서는 `studiofflower.dev`에서 Studio 낙화 포트폴리오 빌더를 사용하는 방법을 정리한 안내서입니다.  
관리자, 승인된 사용자, 방문자가 보는 화면이 서로 다르므로 먼저 권한 구조를 확인해 주세요.

## 1. 접속 주소

- 공개 사이트: `https://studiofflower.dev`
- 관리자 로그인: `https://studiofflower.dev/admin/login`
- 관리자 편집 홈: `https://studiofflower.dev/admin`
- 승인된 사용자 편집 주소: `https://studiofflower.dev/{사용자이름}/editor`
- 승인된 사용자 프로젝트 편집: `https://studiofflower.dev/{사용자이름}/editor/projects`
- 승인된 사용자 아카이브 편집: `https://studiofflower.dev/{사용자이름}/editor/archive`
- 내 프로젝트: `https://studiofflower.dev/{사용자이름}/editor/studio-projects`

관리자는 `/admin`을 사용하고, 승인된 사용자는 자신의 사용자 이름이 들어간 `/사용자이름/editor` 주소를 사용합니다.

## 2. 계정과 로그인

### 관리자

1. `https://studiofflower.dev/admin/login`으로 이동합니다.
2. 관리자 비밀번호로 로그인합니다.
3. 로그인 후 `/admin`으로 이동합니다.
4. 상단 메뉴에서 `계정 승인`으로 들어가 새 사용자의 신청을 승인하거나 거절할 수 있습니다.

관리자는 전체 콘텐츠와 승인 계정을 관리할 수 있습니다.

### 승인된 사용자

1. `https://studiofflower.dev/admin/login`으로 이동합니다.
2. 계정 신청 탭에서 사용자 이름, 이메일, 비밀번호를 입력합니다.
3. 관리자가 승인할 때까지 기다립니다.
4. 승인 후 같은 로그인 화면에서 이메일과 비밀번호로 로그인합니다.
5. 로그인하면 `/{사용자이름}/editor` 형식의 개인 편집 주소로 이동합니다.

사용자 이름은 영문으로 시작해야 하며, 영문, 숫자, 하이픈, 언더스코어만 사용할 수 있습니다.

## 3. 공개 사이트와 편집 화면의 차이

Studio 낙화는 초안과 게시본을 분리합니다.

- 편집 화면에서 수정한 내용은 초안입니다.
- 초안은 저장해도 공개 사이트에 바로 반영되지 않습니다.
- 방문자가 보는 공개 사이트는 게시된 데이터만 표시합니다.
- 공개 사이트에 반영하려면 반드시 `게시`를 눌러야 합니다.

즉, 안전한 작업 흐름은 다음과 같습니다.

1. 로그인
2. 편집 화면 열기
3. 블록과 내용을 수정
4. 저장
5. 미리보기 확인
6. 게시
7. 공개 주소에서 확인

## 4. 홈 빌더 사용법

홈 빌더는 포트폴리오 첫 화면을 만드는 편집기입니다.

관리자:

```text
https://studiofflower.dev/admin/editor
```

승인된 사용자:

```text
https://studiofflower.dev/{사용자이름}/editor
```

화면은 세 영역으로 나뉩니다.

- 왼쪽: 섹션과 블록 추가
- 가운데: 실시간 미리보기
- 오른쪽: 선택한 블록 또는 섹션 설정

주요 작업:

- 블록 클릭: 해당 블록 선택
- 텍스트 클릭: 바로 입력
- 텍스트 드래그: 글꼴, 크기, 색상, 볼드, 줄 간격 설정
- 블록 테두리 버튼: 복사, 삭제, 옵션 열기
- 블록 사이 `+` 버튼: 원하는 위치에 새 블록 삽입
- 하단 Command Bar: `/` 명령어로 빠르게 블록 추가
- 데스크톱, 태블릿, 모바일 버튼: 반응형 미리보기 전환

## 5. 프로젝트 편집

프로젝트 편집은 프로젝트 목록과 상세 페이지 내용을 관리하는 화면입니다.

관리자:

```text
https://studiofflower.dev/admin/projects
```

승인된 사용자:

```text
https://studiofflower.dev/{사용자이름}/editor/projects
```

할 수 있는 작업:

- 프로젝트 추가
- 프로젝트 제목, 설명, 연도, 역할, 카테고리, 태그 수정
- 표지 이미지 변경
- 상세 페이지 블록 편집
- 프로젝트 순서 드래그 앤 드롭
- 프로젝트 삭제
- 프로젝트별 미리보기 확인

프로젝트 상세 페이지도 홈 빌더처럼 블록 기반으로 편집합니다. 텍스트, 이미지, 갤러리, 임베드, 탭, 버튼, 구분선 등을 사용할 수 있습니다.

## 6. 아카이브 편집

아카이브는 작업 과정, 리서치, 레퍼런스 노트를 정리하는 공간입니다.

관리자:

```text
https://studiofflower.dev/admin/archive
```

승인된 사용자:

```text
https://studiofflower.dev/{사용자이름}/editor/archive
```

할 수 있는 작업:

- 아카이브 페이지 구조 편집
- 노트 목록 섹션 관리
- 텍스트와 이미지 블록 추가
- 탭 내부 블록 편집
- 공개 아카이브 화면 미리보기

## 7. 블록 추가 방법

### Command Bar

편집기 하단의 입력창에서 `/`를 입력하면 명령어 목록이 열립니다.

예시:

- `/#`: 제목 1
- `/##`: 제목 2
- `/###`: 제목 3
- `/####`: 제목 4
- `/text`: 본문
- `/image`: 이미지
- `/gallery`: 갤러리
- `/quote`: 인용
- `/button`: 버튼
- `/divider`: 구분선
- `/embed`: 임베드
- `/contact`: 연락처 섹션

### 블록 사이 `+` 버튼

미리보기 화면에서 블록 사이에 마우스를 올리면 `+` 버튼이 나타납니다.  
이 버튼을 누르면 블록 목록이 열리고, 선택한 위치에 새 블록이 삽입됩니다.

### 탭 블록

탭 블록 안에도 블록을 넣을 수 있습니다.

- 탭 내부 클릭 후 `/` 입력
- 탭 내부 `+` 버튼 사용
- 기존 블록을 탭 안으로 드래그 앤 드롭
- 탭 내부 블록 순서 변경

## 8. 텍스트 편집

텍스트가 있는 블록은 미리보기에서 바로 편집할 수 있습니다.

- Enter: 줄바꿈
- Shift + Enter: 줄바꿈
- `-` 입력 후 Space: 글머리 기호 목록
- `1.` 입력 후 Space: 번호 목록
- 텍스트 드래그: 플로팅 옵션 표시
- 볼드, 색상, 글꼴, 크기, 줄 간격 수정 가능

한글 입력 중에는 조합이 끝난 뒤 저장되는 흐름을 기준으로 작업하는 것이 안전합니다. 긴 글을 입력한 뒤에는 저장 상태가 `저장됨`으로 바뀌었는지 확인해 주세요.

## 9. 이미지 업로드

이미지는 다음 방식으로 넣을 수 있습니다.

- 이미지 URL 입력
- 파일 업로드
- 클립보드 이미지 붙여넣기

지원 형식:

- JPG
- PNG
- WebP
- AVIF
- GIF

SVG는 보안상 업로드하지 않습니다.

이미지가 다른 기기에서 보이지 않는다면 다음을 확인해 주세요.

1. 이미지가 로컬 파일 경로가 아닌 `/uploads/...` 또는 `https://...` 주소인지 확인
2. Render 또는 운영 서버에 DB 저장이 연결되어 있는지 확인
3. 업로드 직후 저장을 눌렀는지 확인
4. 공개 사이트에 반영하려면 게시까지 완료했는지 확인

## 10. 저장과 게시

편집 화면에는 저장 상태가 표시됩니다.

- 저장 중
- 저장됨
- 저장 실패
- 충돌 발생

권장 작업 방식:

1. 내용을 수정합니다.
2. 자동 저장 또는 수동 저장을 기다립니다.
3. 저장됨 상태를 확인합니다.
4. 공개에 반영할 준비가 되면 게시합니다.
5. 공개 주소를 새로고침해서 확인합니다.

여러 브라우저 탭이나 여러 기기에서 같은 프로젝트를 동시에 수정하면 충돌이 날 수 있습니다.  
충돌 메시지가 뜨면 최신 버전을 다시 불러오거나, 현재 작업을 내보낸 뒤 비교해 주세요.

## 11. 내 프로젝트

내 프로젝트 화면에서는 편집 상태를 프로젝트 단위로 저장하고 다시 불러올 수 있습니다.

관리자:

```text
https://studiofflower.dev/admin/studio-projects
```

승인된 사용자:

```text
https://studiofflower.dev/{사용자이름}/editor/studio-projects
```

기능:

- 현재 작업 저장
- 프로젝트 열기
- 이름 변경
- 복제
- 삭제
- 내보내기
- 가져오기
- 게시 기록 저장

내보내기 파일 확장자는 다음과 같습니다.

```text
.studiofflower.json
```

이 파일은 다시 편집 가능한 구조화 JSON입니다. HTML 결과물만 저장한 파일이 아니므로, 다른 브라우저나 기기에서 가져와 다시 편집할 수 있습니다.

## 12. 공개 주소

게시할 때 설정한 이름은 공개 주소에 사용됩니다.

예시:

```text
https://studiofflower.dev/jeongminoh-portfolio
```

현재 시스템의 공개 주소 suffix는 `portfolio`입니다.  
예전 `portfoilo` 주소도 기존 게시물을 위해 읽기 호환을 유지하지만, 새로 게시할 때는 `portfolio` 주소를 사용합니다.

프로젝트 상세 페이지 예시:

```text
https://studiofflower.dev/jeongminoh-portfolio/projects/webdevelop
```

## 13. 로그아웃

상단 오른쪽의 `로그아웃` 버튼을 누르면 현재 세션이 종료됩니다.  
공용 기기에서는 작업 후 반드시 로그아웃해 주세요.

## 14. 문제 해결

### 로그인했는데 `/admin`으로 이동합니다.

관리자 계정은 `/admin`을 사용합니다.  
승인된 사용자 계정은 `/{사용자이름}/editor`로 이동해야 합니다. 계속 `/admin`으로 이동한다면 로그아웃 후 승인된 사용자 이메일로 다시 로그인해 주세요.

### 저장이 실패합니다.

다음 순서로 확인해 주세요.

1. 인터넷 연결 확인
2. 새로고침 전 현재 작업 내보내기
3. 다시 로그인
4. 최신 프로젝트 불러오기
5. 다시 저장

### 다른 기기에서 이미지가 안 보입니다.

이미지가 브라우저 임시 URL이나 로컬 파일 경로로 저장되면 다른 기기에서 보이지 않습니다.  
파일 업로드를 사용해 `/uploads/...` 주소로 저장한 뒤 다시 저장하고 게시해 주세요.

### 게시했는데 공개 사이트가 바뀌지 않습니다.

1. 저장만 한 상태인지 확인합니다.
2. 게시 버튼을 눌렀는지 확인합니다.
3. 공개 주소에서 강력 새로고침을 합니다.
4. 다른 공개 주소를 보고 있지 않은지 확인합니다.

### 여러 탭에서 작업하다 충돌이 납니다.

더 최신 저장본이 서버에 있다는 뜻입니다.  
현재 내용을 잃지 않으려면 먼저 내보내기를 한 뒤 최신 버전을 불러와 주세요.

## 15. 운영자 체크리스트

운영 환경에서는 다음 값이 설정되어 있어야 합니다.

```text
DATABASE_URL
STUDIO_ARCHIVE_ADMIN_PASSWORD
STUDIO_ARCHIVE_AUTH_SECRET
```

이미지를 장기 보관하려면 DB 저장 또는 영구 스토리지를 사용해야 합니다.  
Render의 임시 디스크만 사용하면 재배포나 재시작 후 파일이 사라질 수 있습니다.

## 16. 권장 작업 흐름

가장 안전한 흐름은 다음과 같습니다.

1. 로그인
2. 내 프로젝트에서 작업 열기
3. 홈, 프로젝트, 아카이브 편집
4. 저장 상태 확인
5. 내 프로젝트에서 현재 작업 저장
6. 공개 전 미리보기 확인
7. 게시
8. 공개 주소 확인
9. 중요한 변경 후 `.studiofflower.json`으로 내보내기
